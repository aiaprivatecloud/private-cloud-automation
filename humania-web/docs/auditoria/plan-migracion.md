# Plan de migración — HUMANía Web

Este documento define el plan inicial para migrar la web actual hacia el nuevo diseño de HUMANía Web.

## Objetivo

Migrar hacia el nuevo tema y plugin sin perder contenido, datos ni funcionalidades importantes.

## Fase 1: auditoría

- inventariar plugins
- revisar contenido
- revisar suscriptores
- revisar formularios
- revisar SEO
- revisar menús
- revisar dependencias

## Fase 2: staging

- preparar entorno de prueba
- hacer backup
- instalar tema
- instalar plugin
- comprobar errores
- revisar contenido actual con el nuevo tema
- revisar panel del plugin

## Fase 3: adaptación

- crear plantillas necesarias
- corregir estilos
- adaptar home
- adaptar entradas
- adaptar páginas
- adaptar archivo
- revisar accesibilidad
- revisar SEO

## Fase 4: reducción de plugins

- mantener plugins críticos
- sustituir plugins prescindibles
- eliminar plugins no usados
- documentar decisiones
- probar después de cada cambio

## Fase 5: producción

- backup completo
- despliegue controlado
- activación del tema
- activación del plugin
- comprobación posterior
- rollback preparado

## Reglas

- no tocar producción antes de staging
- no desactivar plugins en bloque
- no borrar contenido sin backup
- no cambiar URLs sin plan
- no activar automatización real al inicio
- no publicar automáticamente noticias IA

## Pendiente

- completar inventario real
- decidir plugins mínimos
- preparar checklist de instalación en staging
- preparar checklist de producción
