# Suscriptores y newsletter — HUMANía Web

Este documento servirá para analizar los suscriptores actuales y preparar una futura newsletter de HUMANía.

## Objetivo

Identificar dónde están almacenados los suscriptores actuales, qué consentimiento existe y cómo se podría implementar una newsletter de forma segura y legal.

## Preguntas clave

- ¿Dónde están guardados los suscriptores?
- ¿Qué plugin o sistema los gestiona?
- ¿Qué aceptaron exactamente?
- ¿Aceptaron recibir correos?
- ¿Aceptaron una newsletter?
- ¿Existe fecha de alta?
- ¿Existe origen del consentimiento?
- ¿Existe sistema de baja?
- ¿Existe política de privacidad actualizada?

## Posibles orígenes

- usuarios de WordPress
- comentarios
- formulario de contacto
- plugin de suscripción
- plataforma externa
- lista importada
- seguidores de podcast
- suscriptores de otra plataforma

## Decisiones posibles

- mantener sistema actual
- migrar a servicio externo
- crear integración propia
- pedir nuevo consentimiento
- separar usuarios de suscriptores
- no usar lista hasta revisar consentimiento

## Reglas básicas

- no enviar newsletter sin base legal clara
- no mezclar suscripción técnica con suscripción comercial/editorial
- incluir baja sencilla
- actualizar política de privacidad
- guardar consentimiento si se implementa newsletter
- evitar importar listas sin revisar origen

## Pendiente

- identificar plugin actual de suscriptores
- revisar campos disponibles
- revisar consentimiento actual
- revisar política de privacidad
- decidir sistema futuro de newsletter
