# Seguridad — HUMANía Web

Este directorio recoge la documentación de seguridad del proyecto HUMANía Web.

La seguridad se incorpora como prioridad desde el inicio del proyecto, especialmente porque la web usará tema propio, plugin propio y automatización editorial basada en fuentes externas e inteligencia artificial.

## Objetivo

Definir las reglas básicas de seguridad para el diseño, desarrollo, despliegue y mantenimiento de HUMANía Web.

## Principios

- Seguridad desde el diseño.
- Mínimo privilegio.
- Validar entradas.
- Escapar salidas.
- No confiar en fuentes externas.
- No publicar automáticamente contenido generado por IA sin revisión.
- No guardar secretos en el repositorio.
- Mantener WordPress, tema y plugin actualizados.
- Usar staging antes de producción.
- Registrar errores y eventos relevantes.

## Prioridad

La seguridad tiene la misma importancia que la accesibilidad, el rendimiento y la automatización.

En HUMANía Web, automatizar no significa delegar el control.
