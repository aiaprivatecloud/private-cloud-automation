# Desarrollo seguro de tema y plugin

Este documento recogerá las reglas de desarrollo seguro para el tema propio y el plugin propio de HUMANía Web.

## Objetivo

Evitar vulnerabilidades comunes durante el desarrollo del tema y del plugin.

## Reglas iniciales

- Validar entradas.
- Sanitizar datos.
- Escapar salidas.
- Usar nonces en formularios.
- Comprobar capacidades de usuario.
- No confiar en fuentes externas.
- No guardar secretos en el repositorio.
- Evitar dependencias innecesarias.
- Mantener el código simple y auditable.
